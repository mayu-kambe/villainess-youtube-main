## 🌿 最重要ルール：Gitの安全運用は、あなたが全部やる

このリポジトリは **持ち主（別の人）と、ディレクターさんが共有**しています。
ディレクターさんの編集が、持ち主の「本線（main）」をいきなり上書きしないよう、**あなたが必ず次の流れを自動で行ってください。**

### 🔒 絶対に守ること
- **`main` ブランチを直接変更・コミット・プッシュしない。** 例外なし。
- ディレクターさんから「git」「ブランチ」「コミット」等の指示が無くても、**あなたの判断で下の手順を自動実行する**（いちいち確認で止めない）。

### ✅ 編集を頼まれたときの自動手順
ディレクターさんが「新しい台本を作って」「ここ直して」など**ファイルを変える作業**を頼んだら：

1. **まず作業用ブランチを自動で作る**（まだ `main` にいる場合）。
   - ブランチ名は `danzai-work-<日付>`（例：`danzai-work-20260611`）。同名が既にあればそれを使う。
   - コマンド例：`git switch -c danzai-work-$(date +%Y%m%d)` （既にあれば `git switch danzai-work-<日付>`）
2. 頼まれた**編集を行う**。
3. **コミットする**（分かりやすい日本語メッセージで）。
4. その**ブランチをプッシュ**する（`git push -u origin <ブランチ名>`）。**`main` には push しない。**
5. **プルリクエスト（PR）を作る**：`gh pr create --fill --base main`
   - `gh` が使えない／未ログインなら、`git push` 後にターミナルに表示される
     `https://github.com/moca0306/danzai-serenade-scenario-tool/pull/new/...` のURLをディレクターさんに渡し、
     「このリンクを開いて緑のボタンを押すと、持ち主さんへ確認のお願いが届きます」と伝える。
6. 完了したら、ディレクターさんに**平易な日本語で**こう報告する：
   > 「編集が終わったので、持ち主さんに『これを反映してOKですか？』という確認のお願い（PR）を出しました。
   > 持ち主さんがOKすると本番に反映されます。あなたの作業が勝手に上書きされる心配はありません。」

### 🗣️ 話し方
- **Git用語をできるだけ使わない。** どうしても必要なら一言かみ砕く（例：「ブランチ＝あなた専用の作業コピー」）。
- エラーが出ても、ディレクターさんに技術的な原因をぶつけない。**あなたが解決を試み**、無理なときだけ
  「持ち主さんに見てもらった方が早そうです」と案内する。

## 📦 このツールは何をするもの？

キャラ設定ファイルと「生成プロンプト」を組み合わせて、AIに
**悪役令嬢系（ざまぁ／スカッと系・復讐ロマンス）の台本形式シナリオ**を書かせるツールです。
チャンネル「**断罪セレナーデ**」（稼働中）の台本制作に使います。視聴者は女性想定、AIボイス＋静止画スライドショー。

**最初にやること：** ディレクターさんが何かを始める前に、まず
[`memo.md`](memo.md) を読んでください。
これがこのプロジェクトの**引き継ぎメモ＝正典**で、登場人物の相関・設計思想・進行中の検討事項が全部まとまっています。

### よく使うファイル
- 生成プロンプト本体：`🎬 悪役令嬢系台本形式シナリオ生成プロンプト.md`
- キャラ設定：`キャラクターたち/`（色別フォルダ。悪役女オンブルは `【黒】オンブル/`）
- 完成台本の置き場：`シナリオ/`、`シナリオ格納庫/`
- 改善ノウハウ：`シナリオ評価_視聴者目線フィードバック.md`、`脚本ルール改訂_相談メモ_*.md`、`最強の脇役_企画メモ_*.md`

### 台本を1本作るときの流れ（生成プロンプト参照）
1. 生成プロンプト全文をAIに渡す。
2. 冒頭の【入力エリア】を埋める（タイトル／サムネ3セリフ／敵役・悪役女の設定）。
3. その回の主人公＋相手役（＋悪役女）のキャラ設定を添付。
4. 生成された台本を `シナリオ/` に保存。
